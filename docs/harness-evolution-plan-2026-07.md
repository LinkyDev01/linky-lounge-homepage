# 하네스 진화안 — Sonnet/Opus로 Fable급 성과 내기 (2026-07-04)

이번 세션(2026-07-02 ~ 07-04, 레이지데이 북클럽 UX 개편 ~30MB 트랜스크립트)을 전수 분석해,
**상위 모델이 "알아서" 해내던 것들을 명시적 스캐폴딩으로 옮기는** 진화안입니다.
핵심 논리: 이 세션에서 성과를 만든 건 원천 지능보다 **일관되게 유지된 프로세스 규율**이었고,
규율은 문서·도구·절차로 외재화할 수 있다.

## 2. 이번 커밋으로 구현된 것 (1단계)

| 파일 | 역할 |
|---|---|
| `CLAUDE.md` | 항상 로드되는 법전: 철칙 5·프로젝트 지도·디자인 시스템·공유 CSS 지도·환경 함정 14·GAS 계약·커뮤니케이션 규약. 사실은 전부 레포 grep + 트랜스크립트 채굴로 검증됨 (파일:라인 근거) |
| `docs/DECISIONS.md` | 운영자 결정 로그 (34건 + 상시 원칙 6) — append-only, 세션 간 연속성의 원천. "보류 부활 금지"의 근거 문서 |
| `.claude/skills/lazyday-deploy/` | 배포 절차서: 승인 게이트→로컬 검증→커밋 규약→READY 확인→토큰 재발급→curl 검증→새 PR→프로덕션 마커 폴링 |
| `.claude/skills/lazyday-preview-migrate/` | 이식 절차서: 범위 확정→대응물 표→값 그대로 복사→**픽셀 대조(다상태 스크린샷+렌더값)**→고아 정리 |
| `scripts/shot.mjs` | 스크린샷/렌더값 검증 CLI — Playwright 보일러플레이트 제거. `--selector`·`--click`(다중)·`--eval`·390px 기본. 환경 경로 자동 폴백 |

설계 원칙: **CLAUDE.md는 법(간결·항상 로드), 스킬은 절차(요청 시 로드), DECISIONS.md는 기억(append-only), shot.mjs는 손(도구)**. 중복 최소화 — 같은 내용이 두 곳에 있으면 한쪽이 썩는다.

## 3. 권고 (2단계 — 다음 세션들에서)

1. **Sonnet 스모크 테스트**: 다음 자잘한 UI 수정 1~2건을 의도적으로 Sonnet 세션으로 수행해 보고, 파이프라인 이탈 지점을 관찰 → CLAUDE.md 보강. (측정 기준: 증거 스크린샷 없이 완료 선언했는가 / 보류 항목을 건드렸는가 / 공유 CSS를 확인했는가 / 새 PR을 만들었는가)
2. **SessionStart 훅** (`.claude/settings.json`): 세션 시작 시 `git branch + docs/DECISIONS.md 헤드 10줄 + dev 서버 상태`를 자동 출력하는 스크립트. 문서를 "읽으라"고 하는 것보다 "이미 눈앞에 있는" 쪽이 약한 모델에 확실함. (이번 커밋엔 미포함 — 훅은 실행 권한이라 운영자 확인 후 도입 권장)
3. **PR 템플릿** (`.github/pull_request_template.md`): "실사이트 변경 / 프리뷰 전용 / 검증 내역 / 관련 결정" 4섹션 — 이번 세션 PR 본문 관행의 고정화.
4. **CI 최소 게이트**: GitHub Actions로 `tsc --noEmit` 1개만이라도 — 지금은 Vercel 빌드가 유일한 게이트라 타입 에러가 배포 실패로만 드러난다.
5. **결정 로그 자동화 습관**: 운영자 결정이 나온 턴에서 즉시 DECISIONS.md 커밋에 포함 (배포 커밋에 동승). "세션 끝나고 정리"는 잊힌다.
6. **잔여 리스크 정리** (발견 사항): `gas/interview-booking.gs`에 SOLAPI 키 평문 잔존(공개 레포) — 키 로테이션 + 스크립트 속성 이전 권장. `docs/lazyday-redesign/`은 obsolete 표시 필요.
