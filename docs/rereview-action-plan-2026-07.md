### D2 재설계안 (유일하게 뒤집힌 결정)

숫자 잔여석("2석 남음")이 무너진 이유 3가지:
1. **원천 데이터가 없다** — 신청 폼이 희망 요일을 수집하지 않고, 상품 모델 자체가 '회차별 요일 유동 선택'이라 요일별 잔여석이라는 수치가 정의되지 않음.
2. **법적 리스크** — 하드코딩 값이 방치되면 허위 재고 표시(전자상거래법 다크패턴 규제·표시광고법의 대표 단속 유형).
3. **논리 충돌** — 인터뷰 선별제에서 신청≠좌석 점유. "1석 남음"은 선착순을 암시하는 사실 왜곡이고, 정원 6석에서 "4석 남음"은 역(逆) 소셜프루프.

**대체안**: ① 요일별 3단계 정성 상태(`모집 중 / 마감 임박 / 마감`) — GAS 시트 셀 3개를 운영자가 토글, 사이트는 ISR로 읽음. ② "합류는 신청 순이 아니라 인터뷰 후 확정돼요" 병기. ③ D-day 카운트다운은 사실 서술로 번역: "D-11" → "신청은 7월 13일(월)까지 받아요". 스티키 CTA는 평시 '3기 신청하기', 마감 3일 전부터만 '· 7/13까지' 병기. ④ 긴급감의 정당화 한 줄: "한 자리에서 모두가 말할 수 있도록, 요일마다 자리를 적게 두어요." — 숫자가 재촉이 아니라 철학의 증거가 됨.

### D1 보완 상세
- 스티키 CTA → 폼 직행 경로에서 가격을 폼 작성 '후'에야 보게 되는 잔여 구멍: 스텝퍼 캡션을 "참가비 150,000원 — 지금은 결제하지 않아요"로 수정해 폼 시작 전 1회 통과.
- **취소선 200,000원**: 1·2기에 실제 200,000원 판매 이력(할인 직전 약 20일, 대법원 2019두36001 기준)이 없으면 종전거래가격 허위 표시. 이력이 없으면 "3기 참가비 150,000원" 단독 표기(런칭 가격 서사). '3기 한정'은 4기 인상이 확정일 때만.
- 회당 30,000원 환산 병기(경쟁력 있는 가격은 숨길수록 손해).

## B. 새로 발견된 이슈 (1차 검토·프리뷰가 놓친 것)

### 법률 — 프리뷰와 무관하게 지금도 위반 상태인 것
1. **개인정보처리방침 페이지가 없음** (개보법 §30, 과태료 대상). 동의 문구의 '개인정보처리방침' 링크가 환불 안내(/policy)로 연결됨. → /privacy 신설 필수.
2. **기존 /policy "2회차 이후 환불 불가"** — 8주 과정은 방문판매법상 계속거래(1개월 이상) 소지가 커 '언제든 해지 + 위약금 10% 이내' 해지권을 배제할 수 없음. 권장 통합 문구: "합류 확정 전 취소 무료 / 결제 후 첫 모임 전 전액 환불 / 시작 후 중도 해지 가능(참여 회차 + 총액 10% 이내 위약금 공제) / 최소 인원 미달 취소 시 전액 환불". 세 접점(참가 안내·가격 카드·/policy) 문구 통일.
3. 필수 동의 고지 법정 요소 누락: 거부권·불이익 문구, 수집 항목 '등' 포괄 금지(인터뷰 답변·인스타·추천인까지 명시), 미합류자는 "확정 통보 후 지체 없이 파기".
4. 4기 알림 폼: 수집·이용 고지 + 광고성 수신 동의 필요("버튼 = 동의 간주" 설계 가능하나 고지문 필수). 발송 문자에 (광고)·수신거부, 야간(21시~) 전송 금지.
5. 추천인 성함 = 제3자 개인정보 → 추천 코드/기수+닉네임 방식 권장.
6. 손글씨 후기: 게재 동의(서면·이니셜 처리) + 실제 후기만(예시 문구 배포 금지), 대가 제공 시 표기.
7. 만 14세 미만 차단 없음 → 참가 자격 '성인 대상' 명시 + 입력값 가드.
8. apply 계열 페이지에 사업자 신원 푸터 없음 → 축약 푸터 추가 권장.

### 전환 — 진짜 병목은 '신청 완료 이후'
9. **희망 요일 문항 부재** (high) — 요일 정원 관리 불가, 인터뷰 20분이 요일 조율로 소모, 마감 요일 희망자가 인터뷰 통과 후에야 거절되는 최악 경험. → 신청 폼에 '주로 참여할 요일(복수 선택)' 추가.
10. **신청 완료→인터뷰 연결이 완료 화면 버튼 1개에 의존** (high) — 이탈 시 복구 경로 없음(sessionStorage는 탭 닫으면 소실, 전화 예약은 7일 제한). → 접수 직후 알림톡/문자로 인터뷰 링크 자동 발송 + 24시간 리마인드 + admin 미완료 목록. 인터뷰 안내는 서비스 이행 연락이라 필수 동의만으로 발송 가능.
11. 완료 화면 보조 CTA('홈으로')가 인터뷰 CTA와 경쟁 → 하단 작은 링크로 강등.
12. 추천인 10% 할인이 placeholder에만 존재하는 비공개 리퍼럴 → 참가 안내·완료 화면에 노출 + 할인 스택 규칙 FAQ 명시.
13. 계측 공백: apply_start, price_card_view, sticky_cta_click, join_info_view 이벤트가 없어 개선 효과 검증 불가 → 반영 전 베이스라인 확보.
14. 카톡 문의 링크가 푸터에만 → FAQ 하단에 추가.

### 기술 — 프리뷰 코드 결함 (정식 반영 전 수정)
15. **D-day 3종(HeroSummary·StickyApplyButtonV2·ClosingSectionV2)이 렌더 중 Date.now() 호출** — 정적 프리렌더에 빌드 시점 D-day 박제 + hydration mismatch. → useDeadline() 훅 일원화(마운트 후 계산, null 동안 '3기 모집 중' 폴백, 60초 갱신).
16. **마감 후 스티키 CTA가 라벨만 바뀌고 신청 폼으로 링크** + 신청 폼·서버에 마감 가드 없음. → 앵커(#next-season) 분기 + 폼/API 마감 검증.
17. 전화 예약 달력도 렌더 중 Date.now() (원본부터 있던 결함) → mounted 가드.
18. 접근성: 커스텀 라디오·체크박스 :focus-visible 부재, 라디오 그룹 fieldset/legend 부재, 에러 aria 연결 없음, #d2691e 위 흰 소형 텍스트 3.63:1 (AA 미달 — 텍스트용 어두운 토큰 분리), FAQ의 무동작 버튼, '자세히 보기' aria-expanded 누락.
19. 마이그레이션 체크리스트: 스토리지 키(preview 접미사), payload에 privacyConsent+동의 시각 추가(GAS 컬럼 확장), track() 이벤트 복원, 환불 문구 확정 전 배포 금지.
20. 기존 타입 에러 4건 수정안 (tsc 통과 검증됨): `const mockBooked: { start: string; end: string }[] = []` / CTAButton은 `type ButtonProps = ComponentPropsWithoutRef<typeof Button>`.

### 순서의 근거
- Phase 0은 **법적 노출과 데이터 유실**이라 프리뷰 논의와 무관하게 먼저.
- Phase 1은 모집 기간 내 반영해야 효과가 있는 것들.
- Phase 2의 알림톡 자동화가 전 항목 중 기대 전환 효과 1위지만 외부 연동(알림톡 사업자 등록 등)이 필요해 Phase 1 뒤.
- Phase 3은 "매 기수 개발자 없이 돌아가는 모집 기계" 만들기.
