# 레이지데이 북클럽 UX/UI 검토의견 (2026-07)

lazyday-bookclub.com 랜딩과 하위 신청 페이지(신청 폼, 서면 인터뷰, 전화 인터뷰 예약) 전체를 실제 화면(모바일 390px / 데스크톱 1440px)과 코드 기준으로 검토하고, 트레바리·동행클럽(구 넷플연가)·문토를 벤치마킹한 결과입니다.

## 1. 전략·정보구조 레벨

### 1-1. 가격 정보의 부재 (가장 큰 이슈)
- 참가비 150,000원(정가 200,000원)은 `서면 인터뷰`와 `전화 인터뷰 예약` 페이지에서 **처음** 노출됩니다. 랜딩에는 FAQ에 "참가비에는 어떤 것들이 포함되어 있나요?"라는 질문만 있고 **금액 자체가 없으며**, 신청 폼(/lazyday/apply)에도 없습니다.
- 사용자는 금액을 모른 채 개인정보를 제출하게 되고, 인터뷰 단계에서야 가격을 보게 됩니다. "숨겨진 가격"은 신뢰를 깎는 대표적 패턴입니다.
- 벤치마크: 트레바리는 상세페이지에 `4개월 310,000원` + `월 77,500원` 환산 + 무이자 할부 안내를 함께 표기. 동행클럽은 모임 카드 단계부터 총액 노출.
- **제안**: 랜딩 '일정·장소' 섹션(또는 별도 '참가 안내' 박스)에 참가비·포함 내역(공간/진행/다과)·환불 규정 1줄을 노출하고, 신청 폼의 제출 버튼 근처에 "참가비 150,000원 · 인터뷰 후 합류 확정 시 결제" 요약을 반복.

### 1-2. 신청 여정이 불투명
- 실제 여정: 신청 폼 → 완료 → 인터뷰(서면 작성 or 전화 예약) → "개별 연락" → (결제·합류). 그런데 이 전체 그림을 **어디서도 미리 보여주지 않습니다.** 완료 화면에서야 다음 단계가 등장합니다.
- 인터뷰제 자체는 동행클럽의 '웰컴 숏터뷰'처럼 멤버 품질 보증 장치로 훌륭한 차별화입니다. 문제는 과정 자체가 아니라 **예고 없음**입니다.
- **제안**: 신청 페이지 상단에 4단계 스텝퍼(① 신청서 작성 → ② 인터뷰(전화 20분/서면 6문항) → ③ 합류 확정 연락 → ④ 참가비 결제)와 예상 소요 기간을 표기. 랜딩 FAQ '인터뷰는 왜 하나요?'에도 동일 그림 연결.

### 1-3. 소셜 프루프 전무
- 랜딩에 후기·멤버 구성·운영 이력 등 신뢰 요소가 없습니다("백 번의 독서모임" 문구가 유일). `/lazyday/review`로 1기 후기를 **수집하고 있으면서 보여주는 곳이 없습니다.**
- 30평 직영 공간(링키라운지)이 핵심 셀링포인트인데 랜딩에 **사진이 한 장도 없습니다** (텍스트 설명뿐).
- 벤치마크: 트레바리는 "15만 명" 헤드라인 + 멤버 구성 차트, 동행클럽은 만족도 99%·참석자 한줄 소개·성비/연령 통계·찜 카운트. 소규모 모임일수록 '누가 오는지 미리보기'가 효과적.
- **제안**: ① 수집된 1·2기 후기 중 2~4개를 '멤버들의 이야기' 섹션으로 랜딩에 노출 ② 공간 사진 3~4장 스트립 추가(lounge 컴포넌트 자산 재활용 가능) ③ 가능하면 기수 멤버 구성(연령대·성비) 간단 표기.

### 1-4. 긴급감·희소성 장치 없음
- 모집 마감일, 정원, 잔여석이 어디에도 없습니다. 7/15 시작인데 "언제까지 신청해야 하는지" 알 수 없습니다.
- 벤치마크: 트레바리 홈 최상단이 '오픈 임박!/마감 임박!' 큐레이션, 동행클럽은 카드에 `11/12` 잔여석과 매진 모임 '팔로우 신청'(다음 시즌 대기 전환).
- **제안**: ① 스티키 CTA 또는 히어로 하단에 "3기 모집 ~7/13" 마감일 표기 ② 요일별 정원·잔여석 표기(소규모라 더 효과적) ③ 마감 후에는 신청 페이지를 '4기 오픈 알림 받기' 폼으로 전환해 대기 수요를 축적.

## 2. 랜딩 페이지 (/lazyday)

### 2-1. 히어로: 무드는 좋으나 정보 전달이 0
- 타이포 포스터는 브랜드 아이덴티티로 훌륭하지만, 첫 화면에서 **무엇을(철학+고전 독서모임)·언제(7/15~)·어디서(사당)·얼마에** 하는지 아무것도 알 수 없습니다.
- 포스터가 이미지 텍스트라 SEO·스크린리더에도 잡히지 않습니다.
- **제안**: 포스터는 유지하되 바로 아래(또는 오버레이)에 "철학과 고전을 함께 읽는 5주 · 7/15–9/6 · 사당 링키라운지 · 150,000원" 한 줄 요약 추가.

### 2-2. 데스크톱 = 모바일 레터박스
- 1440px에서 전체가 ~600px 칼럼이고 양옆 2/3가 단색 갈색(#b8a38a) 여백입니다. 모바일 퍼스트 전략 자체는 타당하지만(트레바리는 아예 m. 도메인으로 리다이렉트), 현재는 여백이 '미완성'처럼 읽힙니다.
- **제안**: 저비용안 — 여백에 이미 보유한 일러스트 자산(poppy/bauhaus)이나 은은한 텍스처 적용. 중간안 — 콘텐츠 max-width 720~800px 상향 + 데스크톱 타이포 스케일업.

### 2-3. '더보기' 아코디언 과다
- 모임 소개 4개 카드, FAQ 대부분, 인터뷰 참고글까지 핵심 콘텐츠가 거의 전부 접혀 있어 **스크롤만 하는 사용자에게는 가치 제안이 보이지 않습니다.** (첫 카드만 기본 펼침 처리한 배려는 좋습니다.)
- **제안**: 모임 소개 4개 카드는 각 카드의 첫 1~2문장(요지)은 항상 펼쳐두고 세부만 접기. 훑기만 해도 "누구와·뭘 읽고·어떻게·어디서"의 뼈대가 읽히게.

### 2-4. 내비게이션 불일치
- 상단 탭 '모임소개'는 `#about`(브랜드 카피)로 스크롤되는데, 실제 "모임 소개" 타이틀 섹션은 `#feature`이며 탭에 없습니다. 5회차 섹션도 탭에서 접근 불가.
- 스크롤 스파이 오프셋(56px)이 실제 sticky 높이와 어긋나는 구간이 있는지 점검 권장.

### 2-5. 클로징 전환 블록 부재
- 페이지가 로고(BrandCloseSection)로 끝납니다. 감성적 마무리는 좋으나 마지막 전환 기회가 없습니다.
- **제안**: 로고 위에 "3기, 7월 15일 시작 · ~7/13 마감" + 신청 CTA + '다음 기수 알림 받기'를 배치.

### 2-6. 책 소개 섹션 (강점)
- 표지 + 발췌 인용 + '이 책을 고른 이유'는 트레바리 '읽을거리' 탭과 같은 구조로 이 사이트의 가장 잘된 섹션입니다.
- 소소한 제안: 모바일에서 회차당 스크롤이 길어 발췌 1개 + 노트 요약 후 더보기 고려, 서점 구매 링크 추가.

## 3. 신청 페이지 (/lazyday/apply)

1. **가격·환불 정보 없음** — 1-1과 동일. 제출 버튼 근처 1줄이면 충분합니다. 트레바리는 "결제 당일 23:59까지 전액 환불"을 상세페이지에 직접 노출합니다.
2. **마케팅 동의가 필수** — "마케팅 활용 및 개인정보 수집에 동의합니다 **(필수)**"는 개인정보보호법상 문제가 됩니다. 마케팅 활용 동의는 **선택 동의로 분리**해야 하며, 필수(수집·이용)와 묶어 강제하면 안 됩니다. 체크박스 2개로 분리 + 개인정보처리방침(/policy) 링크 연결 권장.
3. **완료 화면 '돌아가기'가 인스타그램으로 이동** — 라벨과 동작 불일치. 홈(`/lazyday`)으로 보내거나 라벨을 "인스타그램 보러가기"로.
4. **성별 필수 수집에 이유 없음** — "모임 성비 구성을 위해 여쭤봐요" 같은 마이크로카피 한 줄로 거부감을 낮출 수 있습니다.
5. **홈으로 돌아갈 경로 없음** — 로고가 링크가 아니어서 이탈 경로가 브라우저 뒤로가기뿐입니다.
6. 잘된 점: 요일 선택형 일정표를 폼 위에 반복 노출, 인터뷰 방식 선택 시 즉시 설명 표시, 서버 접수 확인 후에만 완료 처리(유실 방지), 에러 필드로 자동 스크롤.

## 4. 서면 인터뷰 (/lazyday/apply/interview/written)

1. **[버그] 제출 실패가 무시됩니다** — `doSubmit()`의 `catch {}`가 API 실패를 삼키고 무조건 완료 화면을 보여줍니다. 신청 폼은 서버 확인 후에만 완료 처리하도록 주석까지 달아 고쳐놓았는데, 서면 인터뷰는 같은 유실 리스크가 그대로 남아 있습니다. 동일한 재시도 UX 적용 필요.
2. **진행 도트 불일치** — 도트 6개인데 페이지는 7개(정보 입력 + 질문 6). 1페이지에서 도트가 전부 비활성이라 어색합니다.
3. **부담 완화 장치가 숨어 있음** — 미작성 질문이 있어도 제출 가능한 관대한 정책인데, 그 사실을 마지막 확인 모달에서야 알 수 있습니다. 인트로에 "답하고 싶은 질문만 답해도 괜찮아요 · 약 10~15분"을 명시하면 시작 전환율이 오릅니다. 6문항 전부 장문 에세이형이라 심리적 허들이 큽니다.
4. 잘된 점: localStorage 자동 임시저장(이탈 복구), 신청 정보 자동 프리필, 한 페이지 한 질문 구조, textarea 자동 확장 — 이 페이지의 폼 엔지니어링은 수준급입니다.

## 5. 전화 인터뷰 예약 (/lazyday/apply/interview/schedule)

1. **7일 제한 안내 없음** — 예약은 오늘부터 7일 이내만 가능한데 캘린더는 월 전체를 보여줘 대부분의 날짜가 이유 없이 회색으로 보입니다. "예약은 오늘부터 7일 이내 가능해요" 1줄 추가.
2. 잘된 점: 오늘 첫 가용 슬롯 자동 선택, 선택 시에만 예약 폼 노출, 완료 화면의 '내 캘린더에 추가', 예약 충돌 처리, 프리필. 예약 UX 자체는 잘 만들어졌습니다.
3. '결'·'불균형의 균형' 참고 콘텐츠는 브랜드 차별화 자산인데 인터뷰 예약 페이지 하단에만 있기 아깝습니다. 랜딩(모임 소개/FAQ)에서도 활용을 권합니다.

## 6. 공통·기술

- **SEO**: 서브페이지(apply, interview)의 개별 title/description 점검 필요. 트레바리처럼 JSON-LD(`Event` 또는 `Product`) 스키마 추가 시 검색 노출에 유리.
- **접근성**: 아코디언에 `aria-expanded`가 붙어 있는 점은 좋으나, FAQ의 클릭 가능한 본문 영역(`div onClick`)에는 키보드 접근이 없습니다(FeatureBox에는 있음). 주황 강조 텍스트의 베이지 배경 대비도 일부 4.5:1 미달로 보입니다.
- **후기 페이지(/lazyday/review)**: 익명 허용, 선택 문항 위주 구성은 좋습니다. 수집한 후기의 노출처(랜딩 후기 섹션)를 만들어 수집→활용 루프를 완성하세요.
